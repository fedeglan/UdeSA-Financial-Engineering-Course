# Datasets

This folder contains the core datasets used across the course for financial analysis, asset pricing, backtesting, and empirical research.

The files are stored in **Parquet** format for efficient loading and processing in Python (typically with `pandas` or `polars`).

---

## Files included

- `company_financials.parquet`
- `etf_prices.parquet`
- `spx_prices.parquet`
- `stocks_basic_info.parquet`
- `us_stock_prices.parquet`

---

## 1. `company_financials.parquet`

### Description
Fundamental financial statement data for publicly traded companies.

This dataset contains accounting information reported by each firm, including income statement, balance sheet, and cash flow statement items. It is intended for fundamental analysis, ratio construction, valuation work, and factor research.

### Typical granularity
- **One row per company per reporting period**

### Main identifier
- `symbol`

### Key date fields
- `announce_date`: date the report became public. Use as main date field to avoid lookahead bias.
- `fperiod`: fiscal period label

### Main contents
This dataset includes variables such as:

- **Income statement**
  - `revenue`
  - `cogs`
  - `gross_profit`
  - `sga_expenses`
  - `ebit`
  - `ebt`
  - `net_income`
  - `net_income_parent`
  - `eps_basic`
  - `eps_diluted`

- **Balance sheet**
  - `cash_and_equivalents`
  - `accounts_receivable`
  - `inventory`
  - `current_assets`
  - `ppe_net`
  - `intangibles_net`
  - `noncurrent_assets`
  - `total_assets`
  - `accounts_payable`
  - `short_term_debt`
  - `current_liabilities`
  - `long_term_debt`
  - `noncurrent_liabilities`
  - `total_liabilities`
  - `common_equity`
  - `retained_earnings`
  - `total_equity`

- **Cash flow statement**
  - `operating_cf`
  - `investing_cf`
  - `financing_cf`
  - `capex_total`
  - `dividends_paid`
  - `stock_common_buyback`
  - `debt_net`
  - `stock_net`
  - `net_change_cash`

### Notes
- Some fields may be missing for certain companies or periods.
- Reporting frequency may vary across firms and time.

## 2. `us_stock_prices.parquet`

### Description
Historical price data for U.S. listed stocks.

This dataset contains market prices and trading activity for individual stocks and is the main source for return calculations, volatility estimation, momentum signals, and backtesting.

### Typical granularity
- **One row per symbol per trading day**

### Main identifier
- `symbol`

### Main date field
- `date`

### Main contents
- `open_price`
- `high_price`
- `low_price`
- `close_price`
- `volume_in_units`

### Notes
- Prices are adjusted (contemplate splits, dividends and other events that might distorted them in the past).
- When joining with company-level datasets, use `symbol` carefully since tickers can change over time.

## 4. `etf_prices.parquet`

### Description
Historical price data for exchange-traded funds (ETFs).

This dataset is useful for market regime analysis, sector and style rotation, macro proxies, asset allocation exercises, and tradable benchmark construction.

### Typical granularity
- **One row per ETF per trading day**

### Main date field
- `date`

### Main contents
Columns are labeled as stock symbols. Ex: 'AAPL', 'MSFT', 'A', etc. 
Each column correspond to the close price for that stock in the index's date.

### Notes
- Prices are adjusted.

## 5. `stocks_basic_info.parquet`

### Description
Reference metadata for U.S. stocks.

This dataset contains descriptive and classification information about each company or listed instrument. It is useful for universe construction, filtering, grouping, and enriching analysis outputs.

### Typical granularity
- **One row per symbol**

### Main identifier
- `symbol`

### Main contents
Likely includes fields such as:

- `company_name`
- `exchange`
- `country`
- `sector`
- `industry`
- `sub_industry`
- `description`
- `instrument_type`
- `currency`
- `listing_date`
- `shares_outstanding`

### Important notes
- Fields such as `shares_outstanding` may represent a **snapshot** rather than a full history. However, given there is no historical shares_outstanding dataset, you can assume them constant to calculate market cap in time.

## 6. `fx_rates.parquet`

### Description
Historical foreign exchange rate data for a broad set of currencies.

This dataset is useful for currency conversion, international return normalization, macro-financial analysis, cross-country comparisons, and any workflow that requires mapping local-currency values into a common base currency.

### Typical granularity
- **One row per currency-country pair per month-end**

### Main identifiers
- `currency`
- `country_code`

### Main date field
- `date`

### Main contents
- `currency`
- `country_code`
- `country`
- `fx_rate_per_usd`

### Field interpretation
- `fx_rate_per_usd`: units of local currency per 1 U.S. dollar

### Notes
- Observations are stored at **month-end frequency**.
- This is the main dataset to use when converting local-currency series into USD.
- Some currencies are shared by multiple countries or monetary areas, so the same `currency` may appear with different `country` values.
- Be careful with inversion: this dataset is stored as **local currency per USD**, not USD per local currency.
- Coverage starts vary across currencies and countries.

## 7. `interest_rates.parquet`

### Description
Historical central bank policy rate data for a set of countries/currencies.

This dataset is useful for macro analysis, monetary policy studies, carry-related research, regime detection, and cross-country comparisons of interest-rate environments.

### Typical granularity
- **One row per currency per observation date**

### Main identifier
- `currency`

### Main date field
- `date`

### Main contents
- `currency`
- `country_or_area`
- `policy_rate_pct`

### Field interpretation
- `policy_rate_pct`: policy interest rate expressed in percentage points

### Notes
- This dataset contains **policy rates**, not bond yields, deposit rates, or broad money-market curves.
- Observation frequency is generally **daily**, although the effective change frequency depends on the central bank and country.
- Coverage is not universal across all currencies in the project; some currencies may have no corresponding rate history.
- Start dates vary substantially by country.
- When merging with FX or price datasets, align carefully on `date` to avoid accidental forward-looking joins.

## 8. `vix_prices.parquet`

### Description
Historical price data for the **CBOE Volatility Index (VIX)**.

This dataset contains the daily history of the VIX, which is widely used as a market-based proxy for expected equity volatility and risk sentiment.

### Typical granularity
- **One row per trading day**

### Main date field
- `date`

### Main contents
- `open`
- `high`
- `low`
- `close`

### Notes
- The VIX is an index, not a tradable asset.
- The `close` field is typically the main series used in research and empirical work.
- This dataset is especially useful when combined with `spx_prices.parquet` to study the relationship between market returns and expected volatility.
- Daily VIX data can be used to construct monthly or quarterly predictors by applying end-of-period sampling or aggregation rules, depending on the use case.
- When using the VIX in predictive regressions or backtests, make sure the feature is aligned properly in time to avoid lookahead bias.
